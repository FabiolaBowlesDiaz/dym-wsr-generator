  <action>
Create `scripts/validate_drivers.py` that does the following:

1. **Load DB config** from `.env` using `dotenv` (same pattern as `wsr_generator_main.py`). Instantiate `DatabaseManager` with the config dict.

2. **Get Engine output**: Instantiate `DriversEngine(db_manager, schema='auto')` and call `calculate_by_marca()`. Store the resulting DataFrame as `df_engine`.

3. **Run direct SQL queries** against `fact_ventas_detallado` using the SAME date ranges the engine uses (STD mode):
   - Current period: `{current_year}-{current_month:02d}-01` to `{current_year}-{current_month:02d}-{current_day:02d}`
   - YoY period: `{current_year-1}-{current_month:02d}-01` to `{current_year-1}-{current_month:02d}-{current_day:02d}`
   - Apply the SAME exclusions: `UPPER(marca) NOT IN ('NINGUNA', 'SIN MARCA ASIGNADA')` and `UPPER(ciudad) NOT IN ('TURISMO')`
   - Query for current period grouped by marca:
     ```sql
     SELECT marca,
            COUNT(DISTINCT cod_cliente) AS cobertura,
            COUNT(DISTINCT cuf_factura) AS pedidos,
            ROUND(COUNT(DISTINCT cuf_factura)::NUMERIC / NULLIF(COUNT(DISTINCT cod_cliente), 0), 2) AS hit_rate,
            ROUND(SUM(ingreso_neto_bob) / NULLIF(COUNT(DISTINCT cuf_factura), 0), 2) AS drop_size,
            SUM(ingreso_neto_bob) AS venta_total
     FROM {schema}.fact_ventas_detallado
     WHERE UPPER(marca) NOT IN ('NINGUNA', 'SIN MARCA ASIGNADA')
       AND UPPER(ciudad) NOT IN ('TURISMO')
       AND fecha >= '{current_start}' AND fecha <= '{current_end}'
     GROUP BY marca
     ORDER BY marca
     ```
   - Same query for YoY period.

4. **Compute direct VSLY trends** from the two direct query results: for each brand present in both periods, compute `(current_value / yoy_value) - 1` for cobertura, hit_rate, drop_size.

5. **Merge and compare**: Join `df_engine` with direct query results on `marca`. For each brand, compare:
   - `cobertura`: exact integer match
   - `hit_rate`: match within tolerance of 0.01 (ROUND(,2) rounding differences)
   - `drop_size`: match within tolerance of 0.01
   - `cobertura_trend`: match within tolerance of 0.005
   - `hitrate_trend`: match within tolerance of 0.005
   - `dropsize_trend`: match within tolerance of 0.005

6. **Print comparative table** to console using tabulate or plain formatting:
   ```
   === DRIVER VALIDATION: Marzo 2026 STD (dia 1-{day}) ===

   MARCA          | Metric     | Engine   | Direct   | Diff     | Status
   ---------------+------------+----------+----------+----------+-------
   JOHNNIE WALKER | cobertura  | 1234     | 1234     | 0        | PASS
   JOHNNIE WALKER | hit_rate   | 1.45     | 1.45     | 0.00     | PASS
   JOHNNIE WALKER | drop_size  | 892.33   | 892.33   | 0.00     | PASS
   JOHNNIE WALKER | cob_trend  | -0.0512  | -0.0512  | 0.0000   | PASS
   ...
   ```

7. **Multiplicative identity check** (per user's discretion allowance): For each brand, verify that `cobertura * hit_rate * drop_size` is within 1% of `venta_total` from the direct query. Print a separate section for this check.

8. **Summary line**: Print `X/Y brands passed ALL checks` and overall PASS/FAIL.

The script must be re-runnable (not a one-shot). Use `if __name__ == '__main__':` entry point. Add `sys.path.insert(0, project_root)` so imports from `core/` and `proyeccion_objetiva/` work when run from the scripts/ directory.
